New York Times Archive Visualization

Project Date: March 2024 - Current

Summary

The goal of this project was to build up a large database of New York Times data from the NYT Archive API I wanted to learn how to use SQLite to make and query a database. I thought that the most interesting quantities to visualize from the New York Times Archive were

  • Number of Keywords in an article
  • Abstract Length
  • Article Length
  • Headline Length
  • Frequency of a given search term over time
This project was split up into 2 parts: Building the database and visualizing the data. The code isn't fully cleaned up (ie the key isn't stored safely) or uploaded to github yet at all.

Guiding Questions

  • Have articles gotten longer over time?
  • Have article abstract lengths changed over time?
  • Have article headline lengths gotten shorter over time
  • Are there more keywords in articles today than in any point throughout history?
  • Is Trump Fatigue real? How has the frequency of the search term "Donald Trump" in Abstracts decreased over the last 3 years?

Building The Database

To build the database I decided on the database schema and kept track of the things in each API call that were needed There is a limit of 5 api calls per minute and 500 per second. This allows me to get roughly 40 years of data per day.

Visualization

Most of the guiding questions focused on showing the value of some quantity over time. This meant that I could create graphs with a single function and use it for all the graphs.
The only graph I couldn't make like this is for the frequency of certain search terms because it requires a different SQL query.
